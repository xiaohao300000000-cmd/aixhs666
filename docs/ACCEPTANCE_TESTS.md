# 验收标准

## 1. 基础工程

- `docker compose up` 能启动 API、Worker 和 PostgreSQL
- `/health` 返回正常
- 数据库迁移可以从空库执行
- 测试可以在本地和 CI 中运行
- `.env.example` 不包含真实密钥

## 2. 数据幂等

- 同一内容重复采集不会产生重复内容记录
- 同一评论重复采集不会产生重复评论记录
- 同一内容被多个查询发现时，产生多条发现关系但只有一条内容记录

## 3. 任务系统

- pending 任务可被 Worker 领取
- running 任务超时后可恢复
- 失败任务按上限重试
- 部分成功任务保留已完成数据
- 任务可以手动取消和重跑

## 4. 断点续传

- 搜索分页游标可保存
- 评论分页游标可保存
- Worker 中断后从上次游标继续
- 不从第一页重复执行

## 5. 采集器

- 搜索结果字段缺失时有日志
- 页面结构变化时保存原始快照
- 登录失效时停止该平台任务并报警
- 出现验证页面时不继续高频重试

## 6. 评论区动态预算

- 每批评论计算新增用户率、新表达率和重复率
- 连续两批低新增时停止
- 高价值评论区可以继续展开
- 强制上限生效

## 7. 新词发现

- 可从一批文本中提取候选表达
- 候选表达有代表样本
- 人工批准后可以生成查询
- 新查询有独立产出统计

## 8. 事件雷达

- 事件日历可创建事件
- 事件前后可调整查询优先级
- 事件结束后恢复正常频率

## 9. 需求事件链

- 同一公开账号的相关事件可按时间展示
- 每个事件可追溯原文
- 不同账号不会错误合并

## 10. 飞书

- 高价值信号可推送
- 新词可审核
- 失败任务可查看
- 人工反馈可回写数据库

## 11. 数据导出

- 可以导出查询、内容、评论、公开账号、事件和反馈
- 导出数据保留平台和来源字段
- 原始快照可通过路径查回

## 12. MVP 最终验收

MVP 必须满足：

1. 管理至少 100 个查询
2. 连续运行时单任务失败不影响全局
3. 支持帖子、评论和回复关系
4. 支持增量更新
5. 支持断点续传
6. 支持查询产出统计
7. 支持高价值来源追踪
8. 支持新词发现
9. 支持飞书预警
10. 所有关键功能有测试

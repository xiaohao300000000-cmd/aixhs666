# 开发路线

## 阶段 0：项目地基

目标：不连接真实平台，也能用 Mock 数据跑通完整数据流。

包含：

- 仓库骨架
- FastAPI
- PostgreSQL
- SQLAlchemy
- Alembic
- pytest
- Docker Compose
- 任务状态机
- Mock PlatformAdapter
- 数据去重
- 发现关系
- 基础 API

完成标准：

- 同一内容被多个查询发现时只存一份
- 任务失败可重试
- Worker 重启后可恢复
- Mock 数据可形成帖子、评论、用户、查询和发现关系

## 阶段 1：小红书采集 MVP

包含：

- 复用本地 Chrome 登录状态
- 搜索结果采集
- 详情采集
- 一级评论
- 二级回复
- 原始快照
- 增量采集
- 断点续传

完成标准：

- 单查询稳定执行
- 100 个查询可排队执行
- 单任务失败不影响其他任务
- 内容和评论不重复写入

## 阶段 2：高价值来源与动态采集

包含：

- 高价值帖子池
- 种子账号池
- 竞品账号池
- 评论区动态预算
- 来源评分
- 查询评分
- 增量评论追踪

完成标准：

- 高产来源获得更高采集频率
- 低产来源自动降权
- 评论区可根据新增信息提前停止

## 阶段 3：AI 新词发现

包含：

- 文本清洗
- 低信息文本标记
- Embedding
- 语义聚类
- 新表达发现
- 黑话映射
- 候选查询生成
- 飞书人工审核

完成标准：

- 系统能从真实数据生成新查询
- 可统计新查询的真实产出
- 低产查询可自动暂停

## 阶段 4：事件雷达与需求事件链

包含：

- 教育事件日历
- 事件触发式查询调度
- 同一公开账号的需求事件链
- 信号新鲜度
- 高价值信号预警

完成标准：

- 出成绩、暑假等节点能自动提高相关查询频率
- 同一账号多次相关表达可以按时间排序
- 新鲜信号优先进入飞书预警

## 阶段 5：内容增长闭环

包含：

- 高频需求统计
- 内容选题生成
- 资料包主题建议
- 直播主题建议
- 内容效果回写

完成标准：

- 系统能将高频需求转成内容选题
- 能记录内容发布后的自然咨询或互动结果

## 阶段 6：第二平台

根据阶段 1 至 5 的数据决定第二平台。

候选：

- 抖音
- 知乎
- B站
- 微博
- 搜索引擎和普通网页

选择标准：

- 数据增量
- 平台差异
- 采集稳定性
- 实现成本
- 业务价值

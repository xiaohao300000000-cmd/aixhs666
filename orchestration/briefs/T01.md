# 子任务简报：T01

## 任务名称

仓库骨架。

## 任务目标

建立阶段 0 的最小可运行工程骨架，不连接真实平台，也不实现真实采集逻辑。

本任务必须交付：

- FastAPI 应用骨架
- PostgreSQL 本地开发配置
- SQLAlchemy 2.x 基础连接配置
- Alembic 基础迁移配置，并提供可执行的初始迁移
- pytest 测试入口
- Docker Compose 本地启动配置
- `.env.example`
- `/health` 健康检查接口
- 基础 CI

## 依赖

- 已完成任务：无
- 必须存在的接口：无
- 必须阅读的文档：
  - `WORKER_CODEX_PROMPT.md`
  - `orchestration/briefs/T01.md`
  - `templates/SUBTASK_REPORT.md`
  - `docs/ARCHITECTURE.md` 中的技术栈、模块边界和可恢复性章节
  - `TASKS.md` 中 T01 的目标和验收要求

说明：仓库中部分旧提示词仍提到 `orchestration/packets/TXX.md`。本任务以本简报 `orchestration/briefs/T01.md` 为唯一任务包路径。

## 输入

- 当前仓库只有文档、模板和 orchestration 目录。
- 技术栈来自 `docs/ARCHITECTURE.md`：
  - Python 3.12
  - FastAPI
  - SQLAlchemy 2.x
  - Alembic
  - Pydantic
  - pytest
  - PostgreSQL
- 第一阶段不引入 Redis、Kafka、Kubernetes 或复杂分布式架构。

## 输出

完成后应具备以下能力：

- 本地依赖可安装。
- `docker compose up` 能启动 PostgreSQL 和 API 服务。
- API 暴露 `GET /health`，返回明确的健康状态。
- Alembic 能连接数据库并执行 `upgrade head`。
- pytest 能运行并通过基础测试。
- CI 能在推送或 PR 时安装依赖并运行测试。
- 任务报告写入 `orchestration/reports/T01.md`。

## 验收条件

1. `docker compose up --build` 可以启动服务。
2. `GET /health` 正常返回，例如 `{"status":"ok"}` 或等价 JSON。
3. `pytest` 通过。
4. Alembic 迁移可执行，例如 `alembic upgrade head` 或容器内等价命令通过。
5. 项目结构符合 `docs/ARCHITECTURE.md` 的模块边界。
6. 未实现真实小红书采集、AI 分析、飞书集成或业务评分逻辑。
7. 未修改禁止文件，除任务报告外不改动 orchestration 全局状态文件。

## 测试命令

至少运行并在报告中记录结果：

```bash
pytest
docker compose up --build
curl -fsS http://localhost:8000/health
alembic upgrade head
```

如果命令需要在容器内运行，请在报告中写明实际命令和结果。若本机缺少 Docker 或端口被占用，必须记录为风险或阻塞，并说明已完成的替代验证。

- 需要主控决定的事项

## 停止条件

遇到下列情况立即停止并报告：

- 需要修改禁止文件
- 需要改变已确认技术栈
- 需要新增未在架构文档中出现的重大依赖
- 需要真实平台账号、真实小红书登录态或外部密钥
- Docker、PostgreSQL 或 Alembic 无法完成基础验证
- 无法通过验收测试

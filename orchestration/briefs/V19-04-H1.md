# V19-04-H1 妙搭审核队列继续动作状态热修

## 背景

V19-04 已合并并发布 release `7663164028453293028`。线上只读验收发现：`/leads` 在真实空队列状态显示“如已完成今日目标，可以选择继续审核 20 条”，但“继续审核 20 条”和“只看高优先级”均被禁用。

根因是 `reviewQueueWritesEnabled` 仅允许 `ready`，把已成功加载的 `empty` / `complete` 也当成不可写。正确边界应为：

- 日队列 `ready`、`empty`、`complete`：允许 continue；
- `daily_loading`、`daily_missing`、`daily_unavailable`：禁止 continue；
- Run 模式不展示日队列 continue；
- 任一 mutation pending 时继续禁用全部写动作。

## 唯一目标

修正上述状态合同与页面按钮行为，不扩大 V19-04 范围。

## 允许修改

- `miaoda-console/client/src/features/operator/operator-view-model.ts`
- `miaoda-console/client/src/pages/LeadReviewPage.tsx`（仅在纯合同无法完成时）

- `docs/reports/V19_04_MIAODA_EXPERIENCE_VERIFICATION.md`

## 禁止事项

- 不修改 Python、FastAPI、数据库、BFF、依赖、全局进度文件或外部发布工作区。
- 不使用浏览器，不触发审核、continue、Base/飞书/PostgreSQL 写入，不推送、不合并、不发布。
- 不修改已有 V19-04 业务范围或其他页面。

## TDD 与验收

1. 先补失败测试，证明 `empty` / `complete` 应允许写，loading/missing/unavailable 应禁止写。
2. 最小修复 `reviewQueueWritesEnabled`，保留既有加载/错误禁写回归测试。
3. 运行 `npm test -- --runInBand`、`npm run type:check`、`npm run lint`、`npm run build`、`git diff --check`。
4. 输出 `orchestration/reports/V19-04-H1.md`，记录 RED、GREEN、测试结果和未触发外部动作。
5. 至少两个提交（测试/实现可合并一个，报告单独一个），工作树干净后停止等待主控验收。

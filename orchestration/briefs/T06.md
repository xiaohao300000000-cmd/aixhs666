# 子任务简报：T06

## 任务名称

查询管理 API。

## 任务目标

在 T02 核心数据模型和 T03 任务状态机基础上，实现阶段 0 的查询管理 API。

本任务必须实现：

- 查询 CRUD
- 查询启停
- 查询优先级更新
- 手动运行查询
- 查询统计基础字段

本任务只实现 FastAPI API 层和对应测试，不实现真实平台采集、不实现后台常驻 worker、不修改数据库模型或迁移。

## 输出

完成后应具备：

- 查询创建接口，能创建查询并返回完整查询对象。
- 查询列表接口，至少支持按 `platform`、`status`、`query_type` 过滤。
- 查询详情接口。
- 查询更新接口，支持更新 `query_text`、`query_type`、`status`、`priority`、`source`、`next_run_at`。
- 查询启用接口，将查询状态设为 `active`。
- 查询暂停接口，将查询状态设为 `paused`。
- 查询手动运行接口，为指定查询创建一条 `search` 类型 `CollectionTask`，任务状态为可领取的 `pending`，并把 `query_id`、`platform`、`priority` 和查询文本写入任务。
- 查询统计基础字段输出，至少包含 `run_count`、`last_run_at`、`next_run_at`、`discovery_count`、`task_count`。
- `orchestration/reports/T06.md`。

建议 API 形态：

- `POST /queries`
- `GET /queries`
- `GET /queries/{query_id}`
- `PATCH /queries/{query_id}`
- `POST /queries/{query_id}/start`
- `POST /queries/{query_id}/stop`
- `POST /queries/{query_id}/run`
- `GET /queries/{query_id}/stats`

如需调整路径命名，必须保持语义清晰并在报告中说明。

## 验收条件

1. 可以创建查询，默认状态为 `active`，默认优先级为 `0`。
2. 可以列表查询，并按平台、状态和查询类型过滤。
3. 可以读取单个查询，不存在时返回 404。
4. 可以更新查询文本、状态和优先级，并持久化。
5. start/stop 能在 `active` 和 `paused` 之间切换状态。
6. 手动运行能创建一条 `search` 类型采集任务，任务绑定原查询并处于可领取状态。
7. 查询统计接口返回基础统计字段，不需要实现复杂评分。
8. 测试使用 SQLite 内存库或临时数据库，不依赖真实 PostgreSQL 或网络。
9. 不修改数据库模型、迁移、调度状态机、采集器接口和全局进度文件。

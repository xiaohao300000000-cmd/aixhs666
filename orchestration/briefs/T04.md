# 子任务简报：T04

## 任务名称

PlatformAdapter 与 Mock 采集器。

## 任务目标

在 T01 工程骨架和 T02 数据模型基础上，实现阶段 0 的平台采集边界与可测试 Mock Adapter。

本任务必须实现：

- 统一采集数据对象
- `PlatformAdapter` 协议或抽象基类
- Mock Adapter
- 可覆盖查询、内容、评论、用户的测试数据

本任务只实现平台适配器边界和 Mock 数据源，不实现真实小红书采集、不实现去重服务、不实现调度状态机、不实现 API。

## 输入

- T02 已提供查询、内容、评论、公开用户 ORM 模型。
- 当前 `collectors/__init__.py` 只有采集边界占位。
- T04 应产出的统一数据对象需要能表达：
  - 查询命中的内容条目
  - 内容详情
  - 评论和父子评论标识
  - 公开用户资料
  - 分页或游标信息

## 输出

完成后应具备：

- 一个清晰的 `PlatformAdapter` 协议或抽象基类。
- 一组统一采集数据对象，字段能映射到 T02 的核心表。
- 一个确定性的 Mock Adapter，可返回搜索结果、内容详情、评论列表和用户资料。
- 测试证明 Mock 数据可以被映射并提交到现有 ORM 模型中，覆盖查询、内容、评论和用户。
- `orchestration/reports/T04.md`。

## 验收条件

1. `PlatformAdapter` 定义搜索、内容详情、评论和用户资料读取边界。
2. 统一采集对象包含平台、平台侧内容 ID、平台侧评论 ID、平台侧用户 ID、正文、标题、互动计数、发布时间、URL、区域等必要字段。
3. Mock Adapter 输出确定、可重复，测试不依赖网络。
4. 测试覆盖：
   - 查询返回内容结果。
   - 内容详情可读取。
   - 评论列表可读取，至少包含一级评论和一条回复。
   - 用户资料可读取。
   - Mock 数据可映射到 `Query`、`Content`、`Comment`、`PublicProfile` 并在测试数据库中提交。
5. 不实现 T05 去重逻辑：不得处理唯一冲突、文本哈希、多查询发现关系或 upsert。
6. 不修改数据库模型、迁移和全局进度文件。

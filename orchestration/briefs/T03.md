# 子任务简报：T03

## 任务名称

任务状态机。

## 任务目标

在 T02 的 `collection_tasks` 数据模型基础上，实现阶段 0 的任务状态机核心逻辑。

本任务必须实现：

- 任务创建
- Worker 领取任务
- 超时恢复
- 重试
- partial 状态
- cancel

本任务只实现可测试的核心状态转换和调度服务，不实现真实采集器、不启动后台 worker 常驻进程、不实现 API。

- `TASKS.md` 中 collection task 状态包括：
  - pending
  - running
  - completed
  - partial
  - retry
  - blocked
  - failed

## 输出

完成后应具备：

- 一个清晰的 scheduler 状态机模块。
- 创建任务、领取任务、完成任务、标记 partial、取消任务、失败重试、超时恢复的函数或服务类。
- 聚焦状态转换的测试。
- `orchestration/reports/T03.md`。

## 验收条件

1. 状态转换有测试覆盖。
2. Worker 领取任务时只领取可运行任务，并写入 `worker_id`、`started_at`、`status=running`。
3. 失败任务在未超过 `max_attempts` 时进入可重试状态，超过后进入 `failed`。
4. 超时 `running` 任务可恢复为 `retry` 或 `pending`，并保留错误信息。
5. `partial` 和 `cancelled` 状态可被明确设置。
6. 一个任务失败不影响其他任务。
7. 不修改数据库模型和迁移。

# 子任务简报：T07

## 任务名称

小红书搜索采集。

## 任务目标

在 T04 PlatformAdapter、T05 去重入库和 T06 查询管理 API 基础上，实现阶段 1 的搜索结果采集链路。

本任务必须实现：

- 搜索任务执行入口
- 搜索结果 L0 入库
- 搜索分页游标保存
- 原始搜索页快照保存
- 使用 `xhs` 平台标识运行的可测试采集链路

本任务只实现搜索结果采集，不实现内容详情采集、不实现评论采集、不实现断点续传完整策略、不修改数据库模型或迁移。

## 输入

- T06 手动运行接口会创建 `task_type="search"` 的 `CollectionTask`。
- T04 Adapter 的 `search(query_text, cursor, limit)` 返回统一 `SearchPage`。
- T05 `ingest_search_results` 可将搜索结果写入 `contents` 和 `discovery_relations`。
- T02 已有 `Snapshot` 表，可保存原始搜索页快照路径和哈希。

## 输出

完成后应具备：

- 一个 worker 层搜索采集入口，可领取或执行一条 `search` 类型任务。
- 搜索采集入口能读取任务关联的 `Query`，调用传入的 `PlatformAdapter.search`。
- 搜索结果作为 L0 数据入库：
  - 写入或更新 `contents`
  - 写入 `discovery_relations`
  - 不写评论、不补详情
- 搜索页原始快照保存：
  - 将原始搜索页数据序列化为 JSON 文件或等价本地快照
  - 写入 `Snapshot(entity_type="query", entity_id=query.id, snapshot_type="search_page", object_storage_path=..., content_hash=...)`
- 搜索分页游标保存到 `CollectionTask.cursor_json`，至少包含：
  - `next_cursor`
  - `has_more`
  - `limit`
  - `query_text`
- 任务状态更新：
  - 没有后续页时标记为 `completed`
  - 有后续页时可标记为 `partial` 并保留游标
  - 异常时通过 T03 状态机进入 retry/failed，而不是吞掉错误
- 测试使用 `MockPlatformAdapter`，不依赖真实小红书网络。
- 如果当前环境无法验证真实 Chrome 登录态，必须在报告“未完成事项/风险与问题”中明确说明；不得把 Mock 测试描述成真实页面验证。
- `orchestration/reports/T07.md`。

## 验收条件

1. 可处理一条 `search` 采集任务，并调用 Adapter 搜索。
2. 搜索结果写入 `contents`，重复执行不产生重复内容。
3. 搜索结果写入 `discovery_relations`，同一 query/content 重复发现保持幂等。
4. 同一次搜索页保存一条可追溯原始快照记录，快照内容哈希稳定。
5. 搜索分页游标保存到任务 `cursor_json`。
6. 无后续页时任务进入 `completed`；有后续页时任务进入 `partial` 并保留游标。
7. Adapter 或入库异常时任务通过状态机进入 retry/failed，并记录错误。
8. 测试覆盖成功页、分页页、重复执行幂等、异常重试和快照写入。
9. 不修改数据库模型、迁移、查询 API、调度状态机和全局进度文件。

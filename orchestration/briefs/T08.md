# 子任务简报：T08

## 任务名称

小红书详情采集。

## 任务目标

在 T07 小红书搜索采集完成并验收后，实现阶段 1 的帖子详情补全链路。

本任务必须实现：

- 详情任务执行入口
- 从搜索发现内容进入 L1 正文补全
- 作者、发布时间、标签或等价元数据补全
- 互动数据更新
- 原始详情页快照保存
- 使用 `xhs` 平台标识运行的可测试详情采集链路

本任务只实现内容详情采集，不实现评论采集、不实现二级回复、不实现完整断点续传策略、不修改数据库模型或迁移。

## 输入

- T07 已将搜索结果作为 L0 内容写入 `contents`。
- 详情任务使用 `task_type="content_detail"` 或语义等价的任务类型。
- 详情任务的 `target_id` 应指向平台内容 ID，或在 `payload_json` 中明确提供平台内容 ID。
- `PlatformAdapter.get_content(platform_content_id)` 返回统一 `CollectedContent`。

## 输出

完成后应具备：

- 一个 worker 层详情采集入口，可处理一条内容详情任务。
- 详情采集入口能读取任务目标内容 ID，调用传入的 `PlatformAdapter.get_content`。
- 详情结果作为 L1 数据入库：
  - 更新 `contents.body_text`
  - 更新标题、作者、发布时间、URL、地域和互动数据等已有字段
  - 不写评论、不创建回复关系
- 原始详情页快照保存：
  - 将原始详情页数据序列化为 JSON 文件或等价本地快照
  - 写入 `Snapshot(entity_type="content", entity_id=content.id, snapshot_type="content_detail", object_storage_path=..., content_hash=...)`
- 任务状态更新：
  - 成功时标记为 `completed`
  - Adapter 或入库异常时通过状态机进入 retry/failed，而不是吞掉错误
- 测试使用 `MockPlatformAdapter`，不依赖真实小红书网络。
- 如果当前环境无法验证真实 Chrome 登录态，必须在报告“未完成事项/风险与问题”中明确说明；不得把 Mock 测试描述成真实页面验证。
- `orchestration/reports/T08.md`。

## 验收条件

1. 可处理一条详情采集任务，并调用 Adapter 获取内容详情。
2. 详情字段能更新到已有 `contents` 记录，不产生重复内容。
3. 作者公开 ID 能映射或创建 `public_profiles` 身份记录。
4. 同一详情任务重复执行保持幂等。
5. 同一次详情采集保存一条可追溯原始快照记录，快照内容哈希稳定。
6. 成功时任务进入 `completed`。
7. Adapter 或入库异常时任务通过状态机进入 retry/failed，并记录错误。
8. 测试覆盖成功详情、重复执行幂等、异常重试和快照写入。
9. 不修改数据库模型、迁移、查询 API、调度状态机和全局进度文件。

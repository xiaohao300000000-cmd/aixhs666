# 子任务简报：T02

## 任务名称

核心数据模型。

## 任务目标

在 T01 工程骨架上实现阶段 0 的核心数据库模型和 Alembic 迁移。

本任务必须实现：

- `queries`
- `contents`
- `comments`
- `public_profiles`
- `discovery_relations`
- `collection_tasks`
- `snapshots`
- `collection_events`

重点是字段、唯一键、外键、索引和迁移可执行；不得实现任务状态机、采集器、去重服务或 API 业务接口。

## 输入

- T01 已提供 FastAPI、SQLAlchemy、Alembic、pytest 和 CI 骨架。
- `docs/DATA_MODEL.md` 是字段与索引的权威来源。
- 当前 `alembic/versions/0001_initial_schema.py` 是空的初始 marker。

## 输出

完成后应具备：

- SQLAlchemy 2.x ORM 模型定义。
- Alembic 新迁移，创建 T02 的 8 张核心表。
- 唯一约束和索引符合 `docs/DATA_MODEL.md`。
- 数据模型测试覆盖表名、关键字段、唯一键、索引和外键关系。
- `orchestration/reports/T02.md`。

## 验收条件

1. `queries`、`contents`、`comments`、`public_profiles`、`discovery_relations`、`collection_tasks`、`snapshots`、`collection_events` 均有模型和迁移。
2. 唯一键正确：
   - `contents(platform, platform_content_id)`
   - `comments(platform, platform_comment_id)`
   - `public_profiles(platform, platform_user_id)`
3. 关键索引正确：
   - `queries(status, next_run_at)`
   - `comments(content_id, published_at)`
   - `discovery_relations(query_id, discovered_at)`
   - `collection_tasks(status, scheduled_at, priority)`
4. 外键关系正确，评论支持 `parent_comment_id`。
5. Alembic 迁移可在 PostgreSQL 环境执行。
6. 数据模型测试通过。
7. 不实现 T03 任务状态机，不实现 T04 Mock Adapter，不实现 T05 去重逻辑。

## 测试命令

至少运行并在报告中记录结果：

```bash
pytest
alembic upgrade head
```

如果本地没有 PostgreSQL 或 Docker，必须至少运行：

```bash
pytest
alembic upgrade head --sql
```

并在报告中写清楚本地未能执行在线迁移的原因。主控会用 GitHub CI 补充验证 PostgreSQL 在线迁移。
